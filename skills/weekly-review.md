# 周记生成器

## 触发方式

`/weekly-review` 或 `/weekly-review 2026-06-02`

- 无参数时：汇总本周（周一到今天）的所有文档
- 带参数时：以指定日期所在周的周一为起始日，汇总该周所有文档

---

## 配置

- **Obsidian 库路径**：`<你的Obsidian库路径>`（需在 CLAUDE.md 中配置）
- **文档筛选规则**：时间范围内（周一 ~ 周日）**所有** `.md` 文件，不再限制来源。包括：
  - 微信文章笔记（有 frontmatter，文件名 `{关键字}-{主题}-{YYYY-MM-DD}.md`）
  - 个人笔记（可能无 frontmatter，文件名 `YYYY-MM-DD.md` 或其他格式）
- **周记命名规则**：`周记-W{周数}-{YYYY}.md`（如 `周记-W23-2026.md`）
- **周记存放路径**：与文档同一目录

---

## 执行流程

### Phase 1: 确定时间范围

#### 1.1 计算本周日期范围

用 Python 脚本计算：

```bash
python3 << 'PYEOF'
from datetime import datetime, timedelta

# 获取参数（如果有）
import sys
ref_date = sys.argv[1] if len(sys.argv) > 1 else datetime.now().strftime('%Y-%m-%d')

ref = datetime.strptime(ref_date, '%Y-%m-%d')
# 找到本周一
monday = ref - timedelta(days=ref.weekday())
sunday = monday + timedelta(days=6)
week_num = monday.isocalendar()[1]
year = monday.isocalendar()[0]

print(f"START={monday.strftime('%Y-%m-%d')}")
print(f"END={sunday.strftime('%Y-%m-%d')}")
print(f"WEEK_NUM={week_num}")
print(f"YEAR={year}")
print(f"FILENAME=周记-W{week_num:02d}-{year}.md")

# 列出本周所有日期，用于匹配文件
current = monday
dates = []
while current <= sunday:
    dates.append(current.strftime('%Y-%m-%d'))
    current += timedelta(days=1)
print(f"DATES={','.join(dates)}")
PYEOF
```

如果没有参数传入，使用当天日期作为 ref_date。

记录输出：起始日期、结束日期、周数、文件名、日期列表。

---

### Phase 2: 扫描本周所有文档

#### 2.1 用 Python 或 Glob 查找候选文件

在 Obsidian 库路径下，用 `os.listdir` 或 Glob 列出所有 `.md` 文件。

#### 2.2 筛选本周文档

对每个文件，按以下优先级判断日期：
1. 读取 frontmatter，检查 `created` 或 `date` 字段
2. 如果没有 frontmatter 日期字段，从文件名中提取日期（匹配 `YYYY-MM-DD` 模式）
3. 日期在本周范围内的文件全部纳入

**不再检查 `source` 或 `mp.weixin.qq.com`**。所有类型的文档都纳入周记：
- 微信文章存档（有 frontmatter）
- 个人笔记（无 frontmatter，纯文本）
- 日记/日报（任意格式）

#### 2.3 读取所有文档内容

逐个 Read 每篇符合条件的文档，按文档类型提取信息：

**有 frontmatter 的微信文章**，提取：
- 文章标题（`title`）、来源（`author`）、标签（`tags`）
- 摘要内容（`## 摘要` 章节）
- 关键发现列表（`## 关键发现` 章节）
- 一句话总结（`## 一句话总结` 章节）
- 所有 `[[]]` 双链概念

**无 frontmatter 的个人笔记**，提取：
- 标题：使用文件名（去掉 `.md` 后缀）或文档首行（若是 `[[链接]]` 格式则取链接文本）
- 来源/作者：标记为"个人笔记"
- 标签：从内容中推断或留空
- 摘要：文档的前 2-3 句话或核心数据点
- 所有 `[[]]` 双链概念
- 关键数据表格（如有）

---

### Phase 3: 分析与归类

#### 3.1 主题归类

根据文章的标签、内容和关键发现，将文章归入 2-4 个主题类别。常见类别：

- **技术/工程**：编程工具、开发实践、架构设计
- **AI/机器学习**：大模型、训练技巧、应用案例
- **半导体/硬件**：芯片工艺、制造技术、产业动态
- **行业/商业**：公司动态、市场趋势、投资分析
- **学术/研究**：论文解读、方法论、实验技术
- **个人成长**：学习方法、工具使用、效率提升

#### 3.2 提取跨文章洞察

分析本周所有文章之间的关联，找出：
- **共同主题**：多篇文章讨论的同一话题
- **概念串联**：不同文章中出现相同或相关的 `[[]]` 双链概念
- **趋势观察**：本周阅读集中反映的兴趣方向或关注点
- **互补视角**：不同文章对同一问题的不同角度

#### 3.3 汇总本周关键概念

收集所有文章中的 `[[]]` 双链概念，列出本周高频出现的概念。

---

### Phase 4: 生成周记

#### 4.1 周记模板

```markdown
---
type: weekly-review
week: W{周数}
date_range: {周一日期} ~ {周日日期}
articles: {文章数量}
created: {生成日期}
tags: [周记, {本周主要标签1}, {本周主要标签2}]
---

# 📰 周记 W{周数}：{本周一句话概括}

> [!summary] 本周概览
> {本周一共读了 N 篇文章，涵盖 A、B、C 等主题。核心关键词：`[[关键词1]]` `[[关键词2]]` `[[关键词3]]`。}

---

## 📂 本周阅读清单

| # | 文章 | 来源 | 日期 | 一句话总结 |
|---|------|------|------|-----------|
| 1 | [[{文章文件名}|{文章标题}]] | {来源} | {日期} | {一句话总结} |
| 2 | [[{文章文件名}|{文章标题}]] | {来源} | {日期} | {一句话总结} |
| ... | ... | ... | ... | ... |

---

## 🔍 主题深入

### {主题类别1}

{2-4句话概括该主题下所有文章的核心内容，标注关键 [[双链]] 概念}

**相关文章**：[[文章1]]、[[文章2]]

### {主题类别2}

{同上}

**相关文章**：[[文章3]]、[[文章4]]

---

## 🧠 跨文章洞察

### {洞察1：共同主题/趋势}
{分析多篇文章之间的关联，串联 [[概念]]，提炼出单篇文章看不到的见解}

### {洞察2：概念网络}
{本周文章中反复出现的核心概念及其关联关系}

---

## 📌 本周核心概念

> [!note] 高频双链概念
> `[[概念1]]` · `[[概念2]]` · `[[概念3]]` · `[[概念4]]` · `[[概念5]]`

{每个概念用一句话说明为什么本周重要}

---

## 💭 本周反思

> {站在更高视角，用2-3句话总结本周阅读的收获：这周的关注重点是什么？学到了什么？有什么值得深入的方向？}
```

#### 4.2 双链标注规则

周记中的双链标注遵循以下规则：

1. **文章引用**：使用 `[[文件名|文章标题]]` 链接到原始文章笔记
2. **概念复用**：直接使用文章中已有的 `[[]]` 概念，保持命名一致
3. **新增概念**：在跨文章分析中涌现的新洞察，用 `[[]]` 标注为新概念
4. **主题标签**：每个主题类别名称加 `[[]]`，如 `[[AI编程]]` `[[半导体工艺]]`
5. **周记互联**：在周记中引用上周/下周的周记文件名（如果存在）

#### 4.3 写入文件

将生成的周记保存到：

```
<Obsidian库路径>/周记-W{周数}-{年份}.md
```

---

### Phase 5: 确认与反馈

向用户确认：
1. 周记已生成并列出文件名
2. 本周共收录几篇文档（微信文章 X 篇 + 个人笔记 Y 篇），分几个主题
3. 列出本周核心概念（双链节点）
4. 如果本周没有任何文档，友好提示"本周暂无文档"

---

## 特殊情况处理

- **本周无文档**：输出友好提示，不生成空周记文件
- **只有1篇文档**：仍然生成周记，主题归类为单类别，跨文章洞察部分可写"本周文章较少，待后续积累更多关联"
- **文档跨越多周**：按文件日期归入对应周，而非创建日期
- **重复文档**：同名文件只保留一个
- **已有同名周记**：覆盖更新（因为是同一周的总结）
- **个人笔记无标题**：用文件名或首行作标题，标注"个人笔记"作为来源
- **个人笔记无摘要**：提取前 2-3 句话或核心数据点作为摘要

---

## 注意事项

- 所有内容使用**中文**
- 周记是**二次加工**，不是文档摘要的简单拼接——要有综合分析和洞见
- **微信文章和个人笔记同等对待**，都参与主题归类和跨文章分析
- 个人笔记即使内容简短，也要尝试提取概念和放入阅读清单
- 主题归类要自然，不要为了分类而强行拆分
- 跨文章洞察是周记的核心价值，值得多花篇幅
- 语气轻松但专业，像给未来的自己写信
- `[[]]` 双链要和文章笔记中的命名保持完全一致，确保 Obsidian 图谱能正确串联
