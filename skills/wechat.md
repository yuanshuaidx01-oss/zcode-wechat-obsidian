# 微信文章智能存档到 Obsidian

## ⚠️ 最高优先级：日期规则

**每次生成笔记前，必须先执行 `date +%Y-%m-%d` 获取当前系统日期。** 将该日期用于文件名和 frontmatter 中的 `date`/`created` 字段。每次存档必须重新获取，不可沿用上一篇文章的日期，不可凭记忆填写。

违反此规则将导致所有笔记日期错误，修复成本极高。

---

## 触发方式

`/wechat <微信公众号文章URL>`

例如：
- `/wechat https://mp.weixin.qq.com/s/AacH8hxfzUhzvzA1z69NXw`

输入参数：`{input}` = 用户提供的微信公众号文章 URL

---

## 配置

- **Obsidian 库路径**：`<你的Obsidian库路径>`（需在 CLAUDE.md 中配置）
- **笔记命名规则**：`{关键字}-{主题}-{YYYY-MM-DD}.md`
  - 关键字：从文章标题中提取最核心的2-4个词（如"Intel18A"、"AI芯片"）
  - 主题：文章主要领域（如"半导体"、"人工智能"、"投资"）
  - **日期：必须使用当前系统日期（`YYYY-MM-DD`），严禁硬编码或沿用上一条的日期。每次存档前用 `date +%Y-%m-%d` 获取当天日期。**

---

## 执行流程

### Phase 1: 抓取文章

#### 1.1 用 webReader 获取文章元数据

使用 `mcp__web_reader__webReader` 工具抓取文章，参数：
- `url`: 用户提供的 URL
- `return_format`: "markdown"
- `retain_images`: true
- `with_images_summary`: true
- `no_cache`: true

提取：
- 文章标题
- 文章描述/摘要
- 所有可见的图片 URL

#### 1.2 用 Python 抓取完整页面并提取所有图片 URL

微信文章的大部分图片是懒加载的，webReader 无法全部获取。必须用 Python 脚本抓取：

```bash
python3 << 'PYEOF'
import urllib.request, ssl, re

ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE

url = "ARTICLE_URL"
req = urllib.request.Request(url, headers={
    'User-Agent': 'Mozilla/5.0 (iPhone; CPU iPhone OS 17_4 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Mobile/15E148 MicroMessenger/8.0.47(0x18002f2f) NetType/WIFI Language/zh_CN',
    'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8',
    'Accept-Language': 'zh-CN,zh;q=0.9',
})

resp = urllib.request.urlopen(req, context=ctx)
html = resp.read().decode('utf-8')

# 提取所有 mmbiz 图片 URL
mmbiz_urls = re.findall(r'(https?://mmbiz\.qpic\.cn/[^"\'<>\s\\]+)', html)
seen = set()
unique = []
for u in mmbiz_urls:
    u = u.replace('&amp;', '&')
    if u not in seen:
        seen.add(u)
        unique.append(u)

for i, u in enumerate(unique):
    print(f'{i}: {u}')
print(f'TOTAL: {len(unique)}')
PYEOF
```

将 ARTICLE_URL 替换为实际 URL。记录输出的所有图片 URL。

**注意：** 如果遇到"环境异常"验证页面（HTML 中包含"环境异常"字样），说明被微信拦截了。改用上面的微信 User-Agent 即可绕过。

#### 1.3 过滤无关图片

过滤掉以下类型的图片（不是文章内容图）：
- 公众号头像/logo（通常是 png，尺寸小，URL 中含 `mmbiz_png` 且是第一个）
- 装饰性图片（如分割线、箭头等）
- 二维码图片

**保留规则：** 只保留文章正文中的内容图片（通常是截图、图表、技术图等）。

---

### Phase 2: 逐图分析

#### 2.1 批量分析图片

对每张内容图片，使用 `mcp__4_5v_mcp__analyze_image` 工具分析：

参数：
- `imageSource`: 图片 URL
- `prompt`: "详细描述这张图片中的所有内容，包括所有文字、图表标题、数据、技术细节。用中文回答，尽量完整还原图片中的文字信息。"

**并行执行：** 每次并行分析最多 5 张图片以提高效率。

#### 2.2 汇总图片分析结果

对每张图片的分析结果，提取：
- 图片标题/主题
- 核心技术要点（2-3 句话）
- 关键数据或参数

---

### Phase 3: 生成 Obsidian 笔记

#### 3.0 获取当前日期

**在生成笔记前，必须先获取当前系统日期**：

```bash
date +%Y-%m-%d
```

将此日期用于：
- 笔记文件名中的日期部分
- frontmatter 中的 `date` 和 `created` 字段
- 笔记正文"文章信息"中的日期

**⚠️ 每次存档必须重新获取日期，不可沿用上一篇文章的日期。**

#### 3.1 自动生成标签

根据文章内容，自动判断并添加 3-7 个标签。标签体系包括但不限于：

**技术类：** `#半导体` `#芯片` `#AI` `#机器学习` `#前端` `#后端` `#架构` `#逆向工程` `#光刻` `#晶体管`
**行业类：** `#Intel` `#台积电` `#英伟达` `#苹果` `#互联网` `#金融` `#投资`
**内容类：** `#技术分析` `#行业趋势` `#教程` `#深度解读` `#资讯`
**格式类：** `#微信文章` `#图集`

**标签规则：**
- 必须包含 `#微信文章`
- 至少包含 1 个内容类型标签
- 根据文章主题添加 2-5 个相关标签
- 标签用中文，公司名可用英文

#### 3.2 生成笔记文件名

格式：`{关键字}-{主题}-{YYYY-MM-DD}.md`

示例：
- `Intel18A-半导体-2026-06-07.md`
- `大模型训练-AI-2026-05-20.md`
- `量化交易-投资-2026-04-15.md`

#### 3.3 生成笔记内容

笔记必须使用以下模板：

```markdown
---
title: "{文章标题}"
source: "{原文URL}"
author: "{作者/公众号名称}"
date: {YYYY-MM-DD}
tags: [{标签1}, {标签2}, {标签3}, ...]
created: {YYYY-MM-DD}
---

# {文章标题}

> [!info] 文章信息
> - **来源**：{公众号名称}
> - **日期**：{YYYY-MM-DD}
> - **原文链接**：{URL}

---

## 摘要

{用3-5句话概括文章核心内容，包括：文章在讲什么、主要发现/观点、对读者的意义}

---

## 逐图分析

### 图1: {图片主题}

{图片分析内容，包括：
- 图片展示的核心内容
- 关键技术细节或数据
- 专业术语的简明解释}

---

### 图2: {图片主题}

{同上格式}

---

（每张图都按此格式，直到所有图分析完毕）

---

## 关键发现

1. **{发现1标题}**：{详细说明}
2. **{发现2标题}**：{详细说明}
3. **{发现3标题}**：{详细说明}
...

---

## 一句话总结

>{用一句大白话总结这篇文章的核心价值}
```

#### 3.3.1 Obsidian 双链标注（自动执行）

生成笔记内容后，**必须自动为关键概念添加 `[[]]` 双链标注**，以便在 Obsidian 中形成知识图谱的双向链接。

**标注原则：**

1. **只标注值得单独建笔记的概念**——如果一个概念未来可能被多篇笔记引用，就应该加 `[[]]`
2. **每个概念每篇笔记只标注一次**——首次出现时标注，后续重复出现不再标注（保持阅读流畅）
3. **别名语法**：当通俗说法与规范概念不同时，使用 `[[规范概念|通俗说法]]`，例如 `[[会话管理|做完就开新会话]]`
4. **不加 `[[]]` 的情况**：通用动词/形容词、日常用语、虚词、标题中已出现的词、frontmatter 中的词

**必须标注的6类概念：**

| 类别 | 标注规则 | 示例 |
|------|----------|------|
| **工具/产品/技术** | 具体的产品名、框架名、模型名、协议名 | `[[Codex]]` `[[React]]` `[[GPT-4]]` `[[CUDA]]` |
| **核心方法论/原则** | 可迁移的工作方法、设计原则、最佳实践 | `[[先读后写]]` `[[最小改动原则]]` `[[小步实现]]` `[[MVC架构]]` |
| **工作流节点** | 流程中的关键步骤或环节 | `[[上下文]]` `[[验证]]` `[[计划]]` `[[总结]]` `[[风险点]]` |
| **研究/学术概念** | 学术术语、实验方法、评估指标 | `[[论文复现]]` `[[消融实验]]` `[[一致性检查]]` `[[交叉验证]]` |
| **人物/组织/公司** | 人名、公司名、社区名、开源组织 | `[[陈思州]]` `[[Datawhale]]` `[[OpenAI]]` `[[台积电]]` |
| **领域核心术语** | 文章所属领域的专业术语（首次出现时标注） | `[[光刻]]` `[[晶体管]]` `[[量化交易]]` `[[注意力机制]]` |

**一致性规则（跨文章保持命名统一）：**

- 同一概念始终使用同一个规范名称：`[[论文复现]]` 而非 `[[复现论文]]`
- 公司名用官方名称：`[[英伟达]]` 不写成 `[[NVIDIA]]`（除非原文全篇用英文）
- 方法论用动宾结构或名词短语，简短有力：`[[先读后写]]` 而非 `[[在写代码前先阅读项目]]`
- 配置文件保留原名：`[[AGENTS.md]]` `[[package.json]]`

**执行方式：**

在生成最终笔记文本时，直接在 markdown 中嵌入 `[[]]`。不需要额外的后处理步骤——写笔记时就同步完成双链标注。

#### 3.4 保存笔记

将生成的 Markdown 内容写入 Obsidian 库：

```
<Obsidian库路径>/{文件名}.md
```

使用 Write 工具写入文件。

### Phase 4: 确认与反馈

向用户确认：
1. 文件已保存到 Obsidian 库
2. 显示文件名和路径
3. 显示生成的标签列表
4. 询问用户是否需要修改标签或补充内容

---

## 错误处理

- **微信验证拦截**：如果 curl 被"环境异常"页面拦截，改用微信 User-Agent 的 Python 脚本
- **图片分析失败**：如果某张图片无法分析（空白图、损坏图），跳过并记录
- **重复图片**：如果检测到重复图片（内容相同的相邻图），合并为一条分析
- **空白/占位图**：跳过空白图片，不生成分析条目

---

## 注意事项

- 所有分析内容必须用**中文**
- 技术术语首次出现时给出简明解释（一句话）
- 逐图分析要保留原文的专业性，但语言要易懂
- 标签不要超过 7 个
- 文件名不要包含特殊字符（`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`）
